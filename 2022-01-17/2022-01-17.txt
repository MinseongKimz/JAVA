Method Overloading

메소드의 기능이 같다 + 구분(식별) 가능 → 동일한 이름 허용

class 붕어빵
{	
	int 팥;
	int 반죽;
	int 설탕;
	int 소금;
	int 물;
	int 가열시간;

	붕어빵()
	{
		this.팥 = 50;
		this.반죽 = 60;
		this.설탕 = 5;
		this.소금 = 3;
		this.물= 10;
		this.가열시간 =5;
	}
	

	붕어빵(int 팥, int 반죽, int 설탕, int 소금, int 물, int 가열시간)
	{
		붕어빵(); // 생성자 안에서 또다른 생성자 호출 가능

		this.가열시간 = 가열시간;
	}	
}


class 외부클래스
{
	main()
	{
		붕어빵 ob = new 붕어빵();
	}
}





배열

변수가 많을 때 일일이 다쳐서 선언 해야하나..? 
이때 배열을 사용하자. 단 데이터 타입이 배열에서 같아야한다.

동질의 성격
배열의 길이는 고정 // 길이는 방의 갯수.. 
길이가 만약에 늘면 메모리가 만약에 배열뒤에 뭔 내용이 있었으면
그거 지우고 배열 채울꺼 아니니깐.. 

가변 배열은 늘어난 길이에 맞는 메모리 빈 공간을 새로 팜

배열의 번호는 0번부터 시작

int a; a라는 이름에 변수를 선언
int[] a; == int a[];   a 라는 이름을 가진 배열을 선언

main(String[] args) args라는 이름을 가진 문자열타입 배열 생성..

	자료형[] 변수명;
